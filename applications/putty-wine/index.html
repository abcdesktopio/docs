    WM_CLASS

    putty.exe.putty.exe

    The WM_CLASS property (of type STRING without control characters) contains two consecutive null-terminated strings. These specify the Instance and Class names to be used by both the client and the window manager for looking up resources for the application or as identifying information. to get the WM_CLASS property of an application, use the command line wmctrl -lx

    PRE run command are run before the package install command

    ENV WINEARCH=win64
ENV WINEDLLOVERRIDES="mscoree,mshtml="
RUN mkdir -p /composer/bin
RUN curl -Ls -o /composer/bin/putty.exe https://the.earth.li/~sgtatham/putty/latest/w64/putty.exe

    Install the builded image

    Replace the ABCHOST var set to localhost by default to your own server ip address

    ABCHOST=localhost
curl --output putty-wine.d.3.0.json https://raw.githubusercontent.com/abcdesktopio/oc.apps/main/putty-wine.d.3.0.json
curl -X PUT -H 'Content-Type: text/javascript' http://$ABCHOST:30443/API/manager/image -d @putty-wine.d.3.0.json

    Rebuild the image manually

    Download the Dockerfile manually

    Dockerfile for application putty-wine

    wget https://raw.githubusercontent.com/abcdesktopio/oc.apps/main/putty-wine.d

    build the Dockerfile to create a container image

    docker build --build-arg TAG=3.0 -f putty-wine.d -t putty-wine .

    Install the new image

    If you are using containerd as container runtime, use the ctr command line

    If you are not running this bash command on your abcdesktop node Replace the ABCHOST variable set to localhost by default to your own server ip address

    ABCHOST=localhost
docker inspect putty-wine > putty-wine.json
docker image save putty-wine -o putty-wine.tar
ctr -n k8s.io images import putty-wine.tar
curl -X PUT -H 'Content-Type: text/javascript' http://$ABCHOST:30443/API/manager/image -d @putty-wine.json
