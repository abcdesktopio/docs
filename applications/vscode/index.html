    WM_CLASS

    code.Code

    The WM_CLASS property (of type STRING without control characters) contains two consecutive null-terminated strings. These specify the Instance and Class names to be used by both the client and the window manager for looking up resources for the application or as identifying information. to get the WM_CLASS property of an application, use the command line wmctrl -lx

    Desktopfile

    /usr/share/applications/code.desktop

    A .desktop file is a simple text file that holds information about a program. It is usually placed in /usr/share/applications/.

    PRE run command

    PRE run command are run before the package install command

    RUN curl https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor -o /usr/share/keyrings/microsoft-archive-keyring.gpg
RUN echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/microsoft-archive-keyring.gpg] https://packages.microsoft.com/repos/vscode stable main" > /etc/apt/sources.list.d/teams.list
RUN apt update && apt install -y --no-install-recommends code && apt-get clean && rm -rf /var/lib/apt/lists/*
RUN mkdir -p /usr/share/code/extensions && chmod 777 /usr/share/code /usr/share/code/extensions

    Install the builded image

    Replace the ABCHOST var set to localhost by default to your own server ip address

    ABCHOST=localhost
curl --output vscode.d.3.0.json https://raw.githubusercontent.com/abcdesktopio/oc.apps/main/vscode.d.3.0.json
curl -X PUT -H 'Content-Type: text/javascript' http://$ABCHOST:30443/API/manager/image -d @vscode.d.3.0.json

    Rebuild the image manually

    Download the Dockerfile manually

    Dockerfile for application VSCode

    wget https://raw.githubusercontent.com/abcdesktopio/oc.apps/main/VSCode.d

    build the Dockerfile to create a container image

    docker build --build-arg TAG=3.0 -f VSCode.d -t VSCode .

    Install the new image

    If you are using containerd as container runtime, use the ctr command line

    If you are not running this bash command on your abcdesktop node Replace the ABCHOST variable set to localhost by default to your own server ip address

    ABCHOST=localhost
docker inspect VSCode > VSCode.json
docker image save VSCode -o VSCode.tar
ctr -n k8s.io images import VSCode.tar
curl -X PUT -H 'Content-Type: text/javascript' http://$ABCHOST:30443/API/manager/image -d @VSCode.json
