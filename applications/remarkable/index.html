    WM_CLASS

    remarkable.Remarkable

    The WM_CLASS property (of type STRING without control characters) contains two consecutive null-terminated strings. These specify the Instance and Class names to be used by both the client and the window manager for looking up resources for the application or as identifying information. to get the WM_CLASS property of an application, use the command line wmctrl -lx

    Desktopfile

    /usr/share/applications/remarkable.desktop

    A .desktop file is a simple text file that holds information about a program. It is usually placed in /usr/share/applications/.

    PRE run command

    PRE run command are run before the package install command

    RUN apt-get update && apt-get install --no-install-recommends --yes python3-gtkspellcheck wkhtmltopdf python3-markdown yelp && apt-get clean
RUN curl -Ls -o /tmp/remarkable_1.87_all.deb https://remarkableapp.github.io/files/remarkable_1.87_all.deb && apt-get install  --no-install-recommends --yes /tmp/remarkable_1.87_all.deb && apt-get clean && rm -rf /tmp/remarkable_1.87_all.deb && rm -rf /var/lib/apt/lists/*

    Install the builded image

    Replace the ABCHOST var set to localhost by default to your own server ip address

    ABCHOST=localhost
curl --output remarkable.d.3.0.json https://raw.githubusercontent.com/abcdesktopio/oc.apps/main/remarkable.d.3.0.json
curl -X PUT -H 'Content-Type: text/javascript' http://$ABCHOST:30443/API/manager/image -d @remarkable.d.3.0.json

    Rebuild the image manually

    Download the Dockerfile manually

    Dockerfile for application remarkable

    wget https://raw.githubusercontent.com/abcdesktopio/oc.apps/main/remarkable.d

    build the Dockerfile to create a container image

    docker build --build-arg TAG=3.0 -f remarkable.d -t remarkable .

    Install the new image

    If you are using containerd as container runtime, use the ctr command line

    If you are not running this bash command on your abcdesktop node Replace the ABCHOST variable set to localhost by default to your own server ip address

    ABCHOST=localhost
docker inspect remarkable > remarkable.json
docker image save remarkable -o remarkable.tar
ctr -n k8s.io images import remarkable.tar
